# MCP-First Design Workflow

## MCP-First Design Workflow

Design with `packt-ds` components in Figma using three MCP tools — no manual
Plugin API code required.

| Tool | Role |
|---|---|
| `generate_figma_design` | Captures the live Storybook render as a pixel-perfect Figma frame |
| `use_figma` | Places and configures design-system components inside Figma |
| `get_design_context` | Reads the Figma frame back into code (layer tree, variables, styles) |

---

### Step 1 — Start Storybook

```bash
npm run storybook
# Storybook is now live at http://localhost:6006
```

---

### Step 2 — Capture with `generate_figma_design`

Point the tool at any Storybook story URL to import it as a Figma frame:

```
generate_figma_design({
  fileKey: "<your-figma-file-key>",
  url: "http://localhost:6006/?path=/story/<story-id>"
})
```

This renders the story in a headless browser and places it in your Figma file —
exact spacing, colours, and layout from the live components.

---

### Step 3 — Bind design tokens with `use_figma`

Overlay the raster capture with a real Figma frame that uses design-system
variables instead of hardcoded values:

```js
// Import a token from the packt-ds variable collection
const bgVar = await figma.variables.importVariableByKeyAsync(BRAND_BG_VAR_KEY);

// Bind it to a fill — setBoundVariableForPaint returns a new paint object
const bgPaint = figma.variables.setBoundVariableForPaint(
  { type: 'SOLID', color: { r: 0, g: 0, b: 0 } },
  'color',
  bgVar
);
section.fills = [bgPaint];
```

Once the frame is complete, remove the raster reference:

```js
const ref = await figma.getNodeByIdAsync(CAPTURE_FRAME_ID);
ref.remove();
```

---

### Step 4 — Read back with `get_design_context`

Pull the finished frame back into code at any time:

```
get_design_context({
  fileKey: "<your-figma-file-key>",
  nodeId: "<frame-node-id>"
})
```

Returns the layer tree, bound variables, and a screenshot — use it to verify
the Figma frame matches the component implementation, or to regenerate code from
an updated design.

---

### Step 5 — Validate types with the IDE MCP

```
mcp__ide__getDiagnostics()
```

Surfaces TypeScript errors inline — no separate `tsc` run needed.
