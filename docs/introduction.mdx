## Design tokens

Tokens are defined in `tokens/design-tokens.json` (Tokens Studio format) and built into CSS custom properties via Style Dictionary.

```bash
npm run tokens   # regenerate src/tokens/tokens.css + tokens.dark.css
```

Import in your app:

```ts
import 'packt-ds/tokens';        // light mode (default)
import 'packt-ds/tokens/dark';   // dark mode overrides
```

## Breakpoints

The design system uses four breakpoints. All responsive behaviour keys off the **1024px** boundary — Large Desktop and Desktop are identical.

### Usage in CSS

```css
/* Mobile first */
.component { font-size: 14px; }

@media (min-width: 768px)  { .component { font-size: 16px; } }  /* Tablet+ */
@media (min-width: 1024px) { .component { font-size: 18px; } }  /* Desktop+ */
@media (min-width: 1440px) { .component { font-size: 18px; } }  /* Large Desktop — same as Desktop */
```

### Typography breakpoint behaviour

Heading sizes step down at `< 1024px` (Tablet and Mobile share the same scale). Text styles are constant across all breakpoints.

## Token architecture

```
Primitive tokens  →  Semantic tokens  →  Component usage
  Orange.500            bg-brand-selected     Button background
  #f97141               #f97141               (never raw hex)
```

Raw primitive values are never used directly in components. Always reference semantic tokens so the system can theme correctly.
